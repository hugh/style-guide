./03-layout

Structure & Effects

01

Scanlines

Repeating 4px gradient overlay on body::after. Subtle horizontal lines across the entire viewport.

02

Vignette

Radial gradient on body::before. Edges darken like a CRT monitor bezel.

03

Phosphor Glow

text-shadow on headings. Simulates phosphor bleed where electrons hit the screen.

04

Cursor Blink

Step-end animation on .cursor class. Block cursor █ blinks at 1s interval.

05

Container

Max-width 880px, centered, 2rem padding. Same as other systems.

06

Zero Radius

No border-radius anywhere. Pixels don’t curve.

./05-principles

Key Patterns

1

Pure CRT Black

The darkest background of any system (#080808). No color tint — neutral black like a powered-off CRT screen. Surface layers add minimal gray.

2

Phosphor Green Primary

Bright phosphor green (#33ff33) as the sole primary accent. Named for the P1 phosphor coating used in early monochrome monitors. Applied with text-shadow glow.

3

All Monospace

Every font in the stack is monospace. VT323 (pixel CRT) for display, Fira Code (modern) for body, Share Tech Mono (system) for labels. No serif or sans-serif anywhere.

4

Scanline & Vignette Overlay

Two fixed pseudo-elements on body: repeating-linear-gradient scanlines (4px period) and a radial vignette. Both use pointer-events: none to stay non-interactive.

5

Process Table

The signature unique component. A ps-style grid with PID, command, status, CPU, and memory columns. Status colors use the ANSI palette.

6

Command Prompt Labels

Section labels are prefixed with a green $ prompt character via CSS ::before. Section names use file-path style (./01-palette) instead of numbered headings.
