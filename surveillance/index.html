Surveillance

Clinical and menacing. A design system inspired by security monitoring equipment — dark monitor greys, harsh white text, red REC indicators, and multi-camera grid layouts for single-page HTML documents.

02 — Typography

Font Stack

Two monospace Google Fonts. Share Tech Mono for headings gives the surveillance monitor feel, while IBM Plex Mono for body text provides slightly more readable monospace for extended content. Emphasis comes through weight and color, not font variety.

IBM Plex Mono — Body text, h4 titles, labels, tags, metadata

06 — Key Patterns

Design Principles

These patterns define the visual identity of the Surveillance system.

1

Monitor Black, Not Warm

Backgrounds use cool-tinted near-blacks like powered-off CRT monitors. The slight blue-grey undertone evokes the cold glow of a security control room at 3am.

2

REC Red Dominance

The primary accent is a harsh recording-indicator red. It pulses on active elements and draws the eye like a blinking REC light on a surveillance camera.

3

Operational Semantics

Four semantic colors map to surveillance operations: active green (recording), standby amber (motion/tracking), alert red (flagged), system blue (information/archive).

4

All-Mono Typography

Both fonts are monospace. Share Tech Mono for headings gives the security monitor look. IBM Plex Mono for body text provides slightly more readable extended content while maintaining the clinical equipment feel.

5

Timestamp Everything

Timestamps and camera IDs appear on components like real security footage overlays. Every piece of data feels logged, recorded, and traceable.

6

Evidence Chain

Evidence tags, recognition boxes, and structured data grids evoke forensic documentation and chain-of-custody paperwork. Information is clinical and procedural.

7

Signal Interference

Scan-line overlays, static noise dividers, and flickering animations bring the imperfections of analog video surveillance into the digital design language.
